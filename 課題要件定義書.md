## 1. プロジェクト概要

### 1.1 背景

電通クリエイティブ デジタルクリエイティブ職の選考課題として、「新しい父の日体験」のアイデアが提示された。

本課題の評価軸は「テクノロジーを活用した便利さ」ではなく、**「父子の関係性がどう新しく変わるか」**である。この評価軸を最重視し、本企画を設計した。

### 1.2 プロジェクト概要

| 項目 | 内容 |
|---|---|
| プロダクト名 | SAME AGE（セイム・エイジ） |
| プロダクト形式 | フィジカルカードゲーム × アプリ（ハイブリッド体験） |
| 体験の核心 | 父が子と同じ年齢だった頃の記憶・悩み・感情を、父の日に父子が一緒に掘り起こし対話する体験 |
| ターゲット | 反抗期を経て「気まずさ」を抱えたまま大人になった子と、その父 |
| 開催タイミング | 毎年6月第3日曜日（父の日）。子の年齢が1つ上がるごとに新しい体験が生まれる |

## 2. 課題分析と問題定義

### 2.1 なぜ父の日はないがしろにされるのか

以下の3レイヤーで課題を分解した。

| レイヤー | 分類 | 内容 |
|---|---|---|
| 表面的 | 文化・習慣 | 母の日と違い定番の花・プレゼントがない。6月の梅雨期でイベントが少ない |
| 関係性 | 接点の非対称性 | 母は日常的に接点が多い（食事・家事・会話）。父は「稼ぐ存在」として機能的に捉えられやすい |
| **本質** | **父を「知らない」** | **父が何を考え・何が好きで・どんな人間かを子が知らない。知らないから感謝もできない、話せない** |

### 2.2 本質的な問題定義

> **「父親は、子供にとって最も近くにいる他人である」**

家族として同じ屋根の下に暮らしながら、子は父の内面をほとんど知らない。これが「父の日がないがしろにされる」最も本質的な理由である。

### 2.3 負のループ構造
Step 1：父のことを「知らない」
　↓
Step 2：何をすればいいかわからない
　↓
Step 3：何もしない・話さない
　↓
Step 4：関係が薄れる・気まずくなる
　↓
Step 5：さらに父のことを「知らない」まま（→ Step 1 へ）

## 3. ターゲットユーザー定義（ペルソナ）

### 3.1 ペルソナA：子（主たるユーザー）

| 項目 | 内容 |
|---|---|
| 年齢 | 18〜25歳 |
| 状況 | 反抗期を経て、父との間に「気まずさ」が残っている |
| 感情 | 父に怒っているわけではない。ただ、何を話せばいいかわからない |
| 行動 | 父とは必要最低限の会話のみ。父の日も特に何もしない |
| インサイト | 「父のことをよく知らない」と自覚していない場合が多い |
| 共通体験（実例） | 反抗期に親と頻繁に口論。高校受験の進路に反対された。今は落ち着いたが、どこか距離がある |

### 3.2 ペルソナB：父

| 項目 | 内容 |
|---|---|
| 年齢 | 45〜60歳 |
| 状況 | 子が成長するにつれ、何を話せばいいかわからなくなってきた |
| 感情 | 子とまた話したいが、「今さら」という気持ちがある |
| 行動 | 仕事帰りにイヤホンをして家に帰り、テレビを見て就寝 |
| インサイト | 自分の若い頃の話を、子にしたことがほとんどない |

## 4. コアコンセプト

### 4.1 コンセプトステートメント

> **「父が自分と同じ年齢だった頃」を入口に、今の父と初めて対話する体験**

### 4.2 なぜこのコンセプトが強いのか

| 強みの軸 | 説明 |
|---|---|
| 「同い年」という対等性 | 「親→子」の非対称な関係ではなく、同じ年齢の人間として父と向き合える。これが「気まずさ」を解除する鍵 |
| 会話を目的にしない設計 | カードが話題を提供するため、「さあ話そう」という気まずいきっかけが不要。自然に会話が生まれる |
| 毎年進化する継続性 | 子の年齢が1つ上がるごとに「父の〇歳のカード」が新たに生成される。2年目以降も新鮮な体験が続く |
| フィジカルな共同作業 | スマホを置いて、テーブルを挟んで向き合う体験。物理的な場を共有することで、会話が生まれやすい |
| 父の「人間としての姿」の発見 | 夢・恐れ・後悔を聞くことで、父が「機能的な存在」から「自分と同じ悩みを持つ人間」として見えてくる |

## 5. 体験設計（UX フロー）

### 5.1 全体フロー

| フェーズ | 内容 |
|---|---|
| Phase 0：事前準備（アプリ） | 父がスマホアプリで子の今の年齢を入力。AIが「父が同い年だった頃」の時代背景・流行・社会情勢を自動収集。父は記憶の断片（好きだった音楽、悩み、夢など）を自由に入力する |
| Phase 1：父の日当日（カード） | アプリから「今年の5枚のカード」が生成・プリントされる。父と子がテーブルを挟んで座り、交互にカードを引いて質問に答え合う |
| Phase 2：記録と共有（アプリ） | その日の会話の記録をアプリに保存。来年の父の日に向け、「続き」のカードが予告される |
| Phase 3：翌年（継続） | 子が1歳年を重ねると、「父の〇歳」のカードが新たに届く。毎年、まだ知らない父に出会い続ける |

### 5.2 カードゲームの進行ルール

- 5枚のカードを交互に引く
- カードに書かれた質問に「父が答える」「子も同じ質問に答える」を選べる
- 答えたくないカードはスキップOK（強制しない）
- スマホはテーブルに置かない（フィジカルな体験に集中）

### 5.3 関係変化の設計意図

普通の父子関係：父が教える → 子が聞く（非対称）
SAME AGE：同じ年齢の「人間同士」として話す（対称）

「親」ではなく「18歳だった頃の父」という文脈で向き合うことで、年齢・立場の非対称性が一時的に解除される。これが「気まずさ」を自然に乗り越えるための仕掛けである。

## 6. 機能要件

### 6.1 アプリ機能一覧

| # | 機能名 | 詳細 | 優先度 |
|---|---|---|---|
| F-01 | 年齢入力・計算 | 子の現在年齢・父の現在年齢を入力し、「父が子と同い年だった年」を自動算出する | Must |
| F-02 | 時代背景収集（AI） | 算出した年代の社会情勢・流行・ニュースをAIが収集し、記憶の補助線として父に提示する | Must |
| F-03 | 記憶入力（父） | 父が「好きだった音楽・悩み・夢・後悔」等をテキスト／音声で自由入力できる | Must |
| F-04 | カード自動生成 | 入力された記憶と時代背景を基に、AIが今年の5枚のカードを生成する | Must |
| F-05 | カード出力 | 生成されたカードをPDF形式でエクスポートし印刷できる。または画面表示で進行できる | Must |
| F-06 | 会話記録 | 当日の回答をテキスト・音声で記録し、タイムライン形式で保存する | Should |
| F-07 | 来年のプレビュー | 「来年は父の〇歳のカードが届きます」と継続性を予告する | Should |
| F-08 | プッシュ通知 | 父の日の1週間前に「今年のカードを作りましょう」と父にリマインドする | Could |

## 7. 非機能要件

### 7.1 体験品質要件

| 要件 | 内容 |
|---|---|
| 感情的安全性 | 「答えたくない質問はスキップOK」というルールを明示。強制しない設計にする |
| 包括性 | キャッチボール等の身体能力を前提としない。テーブルに座って話せる体験に限定する |
| シンプルさ | 父世代（50〜60代）がスマホ操作で迷わないUI。最大3ステップで当日カードを出力できる |
| 継続性 | 1年目で完結せず、毎年進化する構造。アプリが翌年への橋渡しを自動で行う |
| 記録の永続性 | 当日の会話記録はクラウド保存し、10年後でも振り返れる形式にする |

### 7.2 制約条件

- 父と子が同じ空間にいることを前提とする（遠距離家族への対応はv2以降）
- カードの印刷環境がない場合はアプリ画面での進行をサポートする
- 個人情報（入力された記憶）はローカル優先処理とし、外部送信を最小化する

## 8. テクノロジー要件

### 8.1 活用テクノロジーの全体像

| テクノロジー | 役割 |
|---|---|
| 生成AI（LLM） | 父の記憶の断片から質問カードを生成。時代背景と個人の記憶を組み合わせ、その家族だけのパーソナライズドなカードを出力する |
| 時代背景データ | 入力した年代の出来事・音楽・映画・スポーツをAPIまたはナレッジベースから取得し、父の記憶の補助線として提示する |
| 音声認識（オプション） | 高齢の父がテキスト入力に慣れていない場合、音声入力を代替手段として提供する |
| PDF生成エンジン | 生成されたカードをA6〜A5サイズの印刷用PDFとして出力する |
| クラウドストレージ | 会話記録を年次でアーカイブ。家族の歴史データベースとして10年・20年後も参照できる |

### 8.2 テクノロジー活用の設計思想

> **テクノロジーは「会話の代替」ではなく「会話のきっかけ」として機能させる**

電通ラボのセミナーで指摘された「手段に溺れないようにしたい」という言葉を体現する設計。AIはカード生成のバックエンドにとどまり、父子の対話はフィジカルなカードと声で行われる。スマホは準備段階と記録にのみ使用する。

## 9. カードコンテンツ設計

### 9.1 カードの5つのカテゴリ

| カテゴリ | テーマ | 質問例（子の年齢が18歳の場合） | 設計意図 |
|---|---|---|---|
| DREAM | 夢・志 | 18歳のとき、将来の夢は何でしたか？今の自分に近い夢でしたか？ | 父の「過去の可能性」を知ることで、今の父を多面的に理解する |
| FEAR | 恐れ・不安 | 18歳のとき、一番怖かったことや不安だったことは何ですか？ | 父も同じように不安を持っていたと知ることで、共感が生まれる |
| SECRET | 本音・秘密 | 18歳のとき、親に言えなかった秘密や本音はありましたか？ | 「言えなかったこと」を共有することで、子の反抗期経験とも重なる |
| JOY | 喜び・日常 | 18歳のとき、何が一番楽しかった？毎週何をして過ごしていましたか？ | 「楽しかった父」を知ることで、父を「人間」として見られる |
| REGRET | 後悔・伝言 | 18歳の自分に、今の自分なら何を伝えたいですか？ | 父の答えの中に、子への本音のメッセージが隠れている |

## 10. 評価軸との整合性

### 10.1 課題の評価軸チェック

| 評価軸 | SAME AGEの対応 | 評価 |
|---|---|---|
| 何らかのテクノロジーを活用 | 生成AIによるカード自動生成・時代背景API連携・クラウド記録 | ◎ |
| プロトタイプの制作・提出 | インタラクティブなHTMLプロトタイプを制作済み（実際に動作する） | ◎ |
| ただ便利にするだけではない | AIはバックエンドに徹し、体験の主役は父子の「声」と「カード」 | ◎ |
| **父子の関係性が新しく変わるか（最優先）** | **「他人だった父」が「人間としての父」に変わる瞬間を設計。1年目で終わらず毎年深化する** | **◎** |
| 具体的な課題・ターゲットの設定 | 「反抗期を経て気まずさが残る父子」という実体験ベースの具体的ターゲット | ◎ |

## 12. 補足：制作プロセスの記録

### 12.1 インサイト発見の経緯

本企画は、制作者自身の実体験から出発している。反抗期に父親と頻繁に口論し、高校受験の進路選択でも対立した。今は怒っているわけではないが、「気まずさ」が残っており、何を話せばいいかわからない状態が続いている。

この個人的な体験を起点に、「なぜ父の日がないがしろにされるのか」を構造的に分析した結果、課題の本質が「父を知らない」ことにあると定義した。

### 12.2 棄却した案

| 案 | 棄却理由 |
|---|---|
| 案A：イヤホン遮断デバイス | 家の中でノイキャンイヤホンをしていると話しかけにくくなるという日常の観察から発想。しかし「父の日の体験」というミッションに対してインパクトが弱く、関係性の変化が小さいため棄却 |
| 案B：産学協同でバラを贈る文化 | 父の日には定番の花がないという課題に対するアプローチ。テクノロジー要件との親和性が低く、電通クリエイティブが求める「新しい体験の発明」に届かないため棄却 |
| 案C：センサー付きキャッチボール | 父子が体を動かすことで自然に会話が生まれるアプローチ。父親の年齢・体力によって体験できない場合があり、普遍性に課題があるため棄却 |

### 12.3 電通ラボのセミナーとの接続

- **「まだこの世界にない新しい体験を発明する」** → 父子が「同い年として話す」という体験は、まだ存在しない
- **「手段に溺れないようにしたい」** → AIはバックエンドに限定し、体験の主役は人間の声にした
- **「たった一人のために」** → 制作者自身の父との気まずさを、企画の出発点とした
- **「消えずに残るモノづくり」** → 毎年記録が積み上がる設計で、家族の歴史として残り続ける
